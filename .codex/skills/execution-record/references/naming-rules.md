# Naming Rules

## Task titles
- Prefer the exact user-approved title when available.
- Keep Chinese titles unchanged.
- If inferred, keep the title short and specific.
- Good example: `房源月报链路优化方案 v2`
- Avoid vague titles like `优化任务` or `修复一些问题`.

## Same-day updates
- Reuse the existing task heading.
- Add `Update - YYYY-MM-DD HH:mm` under the same task instead of creating a duplicate task section.

## Files / Areas
- Prefer subsystem labels plus a few key files.
- Example:
  - `monthly statement export`
  - `backend/src/services/pdfJobsWorker.ts`
  - `backend/src/lib/monthlyStatementInvoiceAttachments.ts`
