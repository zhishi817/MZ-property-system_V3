# 订单跨天时间线性能说明

- 算法复杂度
  - 分段构建：O(N) 解析与过滤、O(N log N) 按起止索引排序
  - 轨道放置：O(N) 单次遍历、常数空间增长按最大并发订单数
- 渲染成本
  - 月视图 42 单元格，每格最多 6 条，最坏约 252 节点；常规数据量可在 16ms 帧预算内
- 优化策略
  - 仅渲染当前月份与选中房号的数据，减少无关节点
  - 统一条形高度与层级，避免重排；过渡动画限定为阴影与边框色
  - 当并发订单数超过 12 时限制每格渲染 6 条并提供查看详情入口
  - 大规模数据建议按房号分页或周视图切片，并引入虚拟滚动容器
- 资源复用
  - 轨道映射按订单 ID 复用，无需逐日重算
  - 颜色与样式令牌常量化，避免重复对象分配
